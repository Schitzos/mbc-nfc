- Added baseline UML/system diagrams for architecture and core role flows.

## 2026-04-30

### Changed

- Rebased all documentation on `KDX#1 - Membership Benefit Card (MBC).pdf`.
- Shifted product scope from a generic e-parking NFC prototype to the KDX MBC assessment.
- Added four required app roles:
  - The Station.
  - The Gate.
  - The Terminal.
  - The Scout.
- Changed core persistence model so the NFC card carries member identity, balance, visit status, and last five transaction logs.
- Updated tariff rule to exclusive member tariff of Rp 2.000 per started hour.
- Added Gate simulation mode for past check-in timestamp testing.
- Added Silent Shield requirement for protecting sensitive card data.
- Made Scout inspection explicitly one-tap and read-only.
- Added explicit assessment deliverables: repository, demo capture, documentation, and presentation scope.
- Clarified Signal UI as a required UI design system direction.
- Marked guest flow as out of scope.
- Added business requirements and system requirements sections.
- Broadened scope from parking-only to reusable cooperative activity check-in/check-out, with parking as the first demo activity.
