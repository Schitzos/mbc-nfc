# KDX Membership Benefit Card Definition of Done

## Task Definition of Done

A task is done when:

- Requirement reference is clear.
- Listed task owner has completed or reviewed their responsibility.
- Implementation follows `DESIGN.md`.
- Code stays within the intended Clean Architecture layer.
- Domain/application code does not import React Native or `react-native-nfc-manager`.
- Every changed executable source file has a created or updated unit test, unless an approved exception is documented.
- Tests pass locally.
- Relevant focused tests and coverage checks are run when the task affects executable source.
- Errors are handled safely.
- Security implications are checked.
- Documentation is updated when behavior changes.
- Git state is clean or expected unstaged/generated files are documented.

## Requirement Definition of Done

A requirement is done when:

- It has a design reference.
- It has one or more tasks.
- It has one or more tests.
- It appears in `TRACEABILITY.md`.
- Acceptance criteria pass.

## MBC Flow Definition of Done

An MBC role flow is done when:

- Station can register and top up card data.
- Real card operations clearly state that NFC is required.
- Devices without NFC or with disabled NFC are blocked from real card operations with actionable guidance.
- Station registration generates internal member ID automatically; staff does not type member ID.
- Station, Gate, Terminal, and Scout do not expose the full internal member ID in normal screens.
- Station can show a local device-side ledger summary for audit/reporting.
- Gate can check in to an activity and reject double check-in.
- Gate simulation mode can write a past activity entry timestamp.
- Terminal can calculate activity fee and checkout.
- Terminal rejects double checkout and insufficient balance safely.
- Scout can inspect balance, status, and latest logs without modifying card data.
- Scout inspection works with one NFC tap.
- Transaction logs keep only the latest five records.
- Parking MVP flow is implemented with clean boundaries so future non-parking activities can be added later without changing card/source-of-truth principles.

## NFC Feature Definition of Done

An NFC feature is done when:

- It works on at least one real Android device where NFC write support is available.
- The app detects NFC support before real card scan/read/write actions.
- Unsupported or disabled NFC states show clear user guidance.
- iOS support is tested or explicitly documented if platform limitations apply.
- Scan success, cancel, timeout, and error paths are tested.
- NFC session cleanup is verified.
- Card write failure does not silently corrupt app state.

## Security Definition of Done

A security-sensitive feature is done when:

- Identity and balance are not plain readable NFC text.
- Internal member ID is generated by the system and is hidden or masked in normal UI.
- Local ledger does not expose more identity than required for reporting/audit.
- Payload validation rejects malformed or tampered data.
- Sensitive data is redacted in logs.
- Silent Shield limitations are documented.
- Scout remains read-only.

## Quality Definition of Done

A quality-sensitive delivery is done when:

- Changed-file unit-test rule passes for all feature branches.
- Every approved test exception includes reason, impacted file, owner, and follow-up.

- Automated unit-test coverage across the whole executable repository source is at least 90%.
- SonarCloud analysis is configured and reports coverage.
- SonarCloud quality gate passes, or any temporary exception is explicitly documented.
- `npm audit` reports 0 known vulnerabilities after dependency changes.
- Git commit-message convention is enforced.
- Changed-file lint checks are enforced through Husky or equivalent Git hooks.
- GitHub Actions release automation is documented so merging to `main` can publish the APK to app distribution.

## Documentation Definition of Done

Documentation is done when:

- Requirements, design, tasks, tests, and traceability are aligned with the KDX MBC brief.
- Design changes have a decision record when meaningful.
- Risk changes are recorded in `RISKS.md`.
- User-facing assumptions are explicit.
- Technical Writer / Presentation Specialist has reviewed technical and non-technical clarity.
- External `CHANGELOG.md` includes the update when the changelog is maintained outside the archive.

## Submission Definition of Done

Submission is done when:

- Repository is ready for GitHub or GitLab sharing.
- Branching, merge-request, and promotion rules for `feature/*`, `develop`, and `main` are documented.
- `.gitignore` excludes generated build output, dependency folders, local environment files, logs, and secrets.
- README contains setup/run instructions, NFC device requirement, mock/demo path, and known limitations.
- No sensitive payloads, keys, credentials, or local-only files are committed.
- Working frontend app has no crash in the main demo flows.
- Image or short video demo evidence is ready.
- APK distribution packaging or tester installation instructions are ready.
- `main` merge is documented as the controlled trigger for APK distribution publishing.
- Technical and non-technical documentation are available.
- Presentation covers UI/UX design, software design, software construction, software quality, software deployment, and software security.
- Demo/Release Engineer confirms APK distribution path, run instructions, demo capture, and known limitations are ready.
- Product Owner confirms the submitted scope matches MVP acceptance.

## QA Evidence Definition of Done

A feature is not done until Senior QA has validated the changed feature on Android simulator/device and attached screenshot evidence, unless the PM/Software Architect approves and records an exception.

Final project delivery is not done until Senior QA provides a use-case testing evidence package with screenshots proving the required parking MVP flows work.

## Firebase Distribution Definition of Done

Release automation is not done until push/merge to `main` triggers GitHub Actions and publishes the Android build to Firebase App Distribution, or a documented blocker/deferral is approved.
